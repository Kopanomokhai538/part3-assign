## Project Overview
Food Factory is a well-known fast-food restaurant based in Bayswater, Bloemfontein[cite: 7, 10]. It caters to students, families, and the local community by serving high-quality, fresh, and affordable meals daily[cite: 7, 10]. This multi-page website project provides Food Factory with an accessible digital presence, featuring an online menu, business hours, local address placement, and functional user communication tools[cite: 7, 8, 9, 11].

---

## Website Goals and Objectives
* **Brand Visibility:** Establish a cohesive, clean web presence for the Bayswater community to replace standalone advertising formats[cite: 7, 10].
* **Customer Accessibility:** Provide an instantly queryable structural digital layout of complete items, details, and pricing categories[cite: 11].
* **Asynchronous Inquiry Capture:** Give users direct client-side feedback channels for submissions and dynamic location routing without page reloads[cite: 8, 9].

---

## Key Features and Functionality
* **Semantic Web Architecture (Part 1):** Five distinct, interconnected web layout pages using standard-compliant structural HTML syntax mappings[cite: 7, 8, 9, 10, 11].
* **Responsive Layout Design (Part 2):** CSS formatting driven by relative width allocations and responsive scaling, linking an external cascading file system to support clean viewing transitions across smartphones and workstations[cite: 7, 8, 9, 10, 11].
* **Asynchronous Client-Side Processing (Part 3):** Isolated standalone modular script architectures handling asynchronous form trapping and visual data integrity constraints[cite: 8, 9].
* **Dynamic Location Integrations (Part 3):** Advanced browser execution loading map markers dynamically for localized community routing.

## File and Architecture Blueprint

### Part 1 & Part 2 Framework Summary
1. **`index.html` (Home):** The primary landing portal introducing the brand, core service specialties (burgers, chicken wings, seafood, drinks), and unique value points[cite: 10].
2. **`about.html` (About):** Highlights the establishment's location, target vision, customer service mission, and overall service delivery models[cite: 7].
3. **`menu.html` (Menu):** Displays a highly legible, structured data matrix mapping categories, product names, descriptions, and regional currency pricing tiers[cite: 11].
4. **`enquiry.html` (Enquiry):** Contains structured input form objects for processing customer messages directly inside the browser canvas[cite: 9].
5. **`contact.html` (Contact):** Outlines essential operational logistics including contact details, precise location coordinates, and shift times[cite: 8].

## Part 3 Functionality Implementations

### Form Data Validation & Async Pipelines
* **Isolated Resource Script Execution:** Decoupled scripts (`js/enquiry (1).js` and `js/contact (1).js`) manage targeted event-driven behaviors across distinct document routes[cite: 8, 9].
* **Client-Side Form Interception:** Forms intercept submission actions programmatically, passing text blocks through string inspection filters to prevent empty inputs[cite: 9].
* **Dynamic Content Injection:** Avoids intrusive pop-up alerts by outputting clean feedback alerts inline within the layout DOM structure upon form verification[cite: 9].
* **Asynchronous Submissions (AJAX):** Simulated background network updates keep the webpage interactive, locking the submit buttons during runtime processing and clearing form states on complete cycles[cite: 9].

### Google Maps API Core Operations
* Replaces basic text descriptors with an interactive, responsive map canvas centered over the intersection layout area[cite: 8].
* Configures targeted markers highlighting the **Corner Milner & Waverley Road, Bayswater** storefront location for direct client access[cite: 8].
